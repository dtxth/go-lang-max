# Быстрое руководство: Редактирование названий элементов структуры

## Обзор

Добавлены новые API endpoints для редактирования названий элементов иерархической структуры университета:

- **Университеты**: `PUT /universities/{id}/name`
- **Филиалы**: `PUT /branches/{id}/name`  
- **Факультеты**: `PUT /faculties/{id}/name`
- **Группы**: `PUT /groups/{id}/name`

## Быстрый старт

### 1. Обновить название университета

```bash
curl -X PUT "http://localhost:8083/universities/1/name" \
  -H "Content-Type: application/json" \
  -d '{"name": "Новое название университета"}'
```

### 2. Обновить название группы

```bash
curl -X PUT "http://localhost:8083/groups/482/name" \
  -H "Content-Type: application/json" \
  -d '{"name": "ВБ(асп)-12"}'
```

## Формат запроса

Все endpoints принимают JSON с полем `name`:

```json
{
  "name": "Новое название"
}
```

## Ответы

**Успех (200 OK):**
```json
{
  "message": "university name updated successfully"
}
```

**Ошибка (400/404/500):**
```json
{
  "error": "описание ошибки"
}
```

## Валидация

- ID должен быть корректным числом
- Название не может быть пустым
- Элемент должен существовать в базе данных

## Интеграция с существующей структурой

После обновления названия:
- Изменения сразу видны в `/universities/{id}/structure`
- Обновляется поле `updated_at`
- Связанные элементы остаются без изменений

## Swagger документация

Полная документация доступна по адресу: `http://localhost:8083/swagger/`
